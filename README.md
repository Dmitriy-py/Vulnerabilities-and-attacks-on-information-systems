Анализ результатов:

### Чем отличаются эти режимы сканирования с точки зрения сетевого трафика?

SYN Scan (-sS): Этот режим сканирования (также известный как “half-open scan”) отправляет SYN пакет на целевой порт. Если порт открыт, сервер отвечает SYN/ACK пакетом. Атакующая машина отвечает RST пакетом, чтобы разорвать соединение (поэтому “half-open”). Если порт закрыт, сервер отвечает RST пакетом. Это самый распространенный и “тихий” вид сканирования, поскольку не устанавливает полноценное TCP-соединение.

Трафик: 
Отправляемые SYN пакеты, принимаемые SYN/ACK (для открытых портов) или RST (для закрытых портов), и отправляемые RST пакеты.
FIN Scan (-sF): Этот режим отправляет FIN пакет на целевой порт. По RFC793, если порт закрыт, сервер должен ответить RST пакетом. Если порт открыт, ответ не должен быть отправлен. Этот режим может обходить некоторые брандмауэры и системы обнаружения вторжений (IDS), которые не отслеживают FIN пакеты.

Трафик:
Отправляемые FIN пакеты, принимаемые RST пакеты (для закрытых портов).
Xmas Scan (-sX): Этот режим отправляет пакет с установленными флагами FIN, PSH, и URG. Поведение сервера аналогично FIN сканированию: закрытые порты должны отвечать RST, открытые - нет. Как и FIN сканирование, может обходить некоторые IDS.

Трафик: 
Отправляемые Xmas пакеты, принимаемые RST пакеты (для закрытых портов).
UDP Scan (-sU): Этот режим отправляет UDP пакет на целевой порт. Если порт открыт, то по спецификации UDP, ничего не должно быть отправлено в ответ. Если порт закрыт, сервер должен ответить ICMP port unreachable пакетом. UDP сканирование, как правило, медленнее, чем TCP сканирование, из-за отсутствия надежного протокола подтверждения.

Трафик:
Отправляемые UDP пакеты, принимаемые ICMP port unreachable (для закрытых портов). Отсутствие ответа после нескольких попыток обычно говорит об открытом порте или фильтрации трафика.

Как отвечает сервер?

| Режим сканирования  | Порт открыт                                        | Порт закрыт                                         | Фильтрация                                                                                                    |
| ------------------- | -------------------------------------------------- | --------------------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| SYN                 | SYN/ACK -> RST (соединение сброшено)               | RST                                                 | Отсутствие ответа, SYN/ACK фильтруются (в зависимости от конфигурации firewall)                               |
| FIN                 | Отсутствие ответа                                  | RST                                                 | Отсутствие ответа                                                                                             |
| Xmas                | Отсутствие ответа                                  | RST                                                 | Отсутствие ответа                                                                                             |
| UDP                 | Отсутствие ответа (или ответ приложения)           | ICMP port unreachable                               | Отсутствие ответа, ICMP фильтруются (поведение идентично открытому порту, требуется дополнительная проверка)  |

Дополнительные замечания:

Фильтрация трафика: Брандмауэры и другие устройства безопасности могут фильтровать трафик, используемый этими типами сканирования, что может приводить к неточным результатам. В таких случаях Nmap может сообщить о “фильтрованных” портах.

Надежность: 
Результаты сканирования могут быть не всегда точными, особенно при использовании FIN, Xmas и UDP сканирования. Различные операционные системы и сетевые устройства могут по-разному обрабатывать эти типы пакетов. UDP сканирование в особенности часто бывает неточным.

Стелс: 
FIN и Xmas сканирование традиционно считаются более “скрытыми” по сравнению с SYN сканированием, так как они реже регистрируются системами обнаружения вторжений. Однако, современные IDS системы вполне способны обнаруживать эти типы сканирования.

Вывод:

Различные режимы сканирования Nmap используют различные типы TCP и UDP пакетов для зондирования целевой системы. Анализ трафика Wireshark позволяет детально изучить, как сервер отвечает на эти пакеты, и как Nmap интерпретирует эти ответы для определения состояния портов. Выбор режима сканирования зависит от целей сканирования, сетевой топологии, и конфигурации целевой системы. SYN scan – наиболее надежный и часто используемый, тогда как другие типы сканирования могут быть полезны для обхода некоторых средств защиты или получения информации в специфических ситуациях.
